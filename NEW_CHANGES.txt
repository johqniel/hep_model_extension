Summary of Changes (March 2026 Updates)

I. Simulation Core & Modules

    a. Dynamic Age Tracking @sandesh
       - Added `age_ticks` (1 tick = 1 week) and `age_years` variables to the `Agent` type.         
       - Added `ticks_since_last_birth` to the `Agent` type.
       - These variables naturally increment every tick, removing reliance on old age markers.

      in src/data_structures/mod_agent_world.f95
   
    a2. get agent functions @sandesh
       - Added robust internal utility functions `get_male_from_cell` and `get_female_from_cell` to `world_container`.


    b. Separation of Technical/Permanent vs. Toggleable Modules
       - Created a new module `src/simulation_modules/mod_technical_modules.f95` for operations that must always run.
       - Moved `update_agent_age` into this technical module to ensure age is updated unconditionally.

         update_agent_age also updates ticks_since_last_birth
         in src/simulation_modules/mod_technical_modules.f95

    c. Decoupling Base Density/HEP from Population Pressure
       - Completely separated density/hep calculations from the old Verhulst population pressure model.
       - agent counting, density computation, density smoothing, flow computation 
         is now permanently handled by `update_density_and_hep_grid` in `mod_technical_modules.f95`.

       - `pop_pressure_func` please check if we still need this and if yes how it should be implemented

    a. Removed dead/stale variables from `mod_grid_id.f95`.


IV. LaTeX Documentation Updates

   The pdf documentation files where updated to reflect changes.

V. Clustering Integration & Configuration

    a. Cluster Store Moved into World
       - `cluster_store` is now a member of `world_container` 

    b. All Clustering Parameters Now Configurable
       - Clustering vars are now part of config
       - smoothing radius is now part of config (still called watershed smoothing -> change @ Daniel)

    c. Zero Density Masking
       - no density => no cluster

    d. Migration Tracking Removed
       - Clusters are defined per cell they and then agents in that cell are also in that cluster

    e. Permanent Module
       - Clustering is not a module anymore but always on

    a. Clustering Vars Moved to General Settings
