HEP Clustering Framework Analysis

This document summarizes the current state and proposed enhancements for the spatial clustering system in the HEP simulation. The goal is to provide a unified framework for switching between Grid-based and Agent-based clustering algorithms.

Proposed Algorithms

Grid-Based Existing

1. Watershed Clustering

A gradient-ascent algorithm that segments the landscape into "basins" based on local density peaks.

Density Surface
➔
Local Maxima (Seeds)
➔

Grid Labels
  • Input: Human density grid (smoothed).
  • Mechanism: Every cell climbs the "hill" to the nearest peak.
  • Output: Each grid cell is assigned a cluster ID. Agents inherit the ID of their current cell.
Grid-Based Existing

2. Auto K-Means

A centroid-based clustering algorithm where the number of clusters (K) is automatically detected using local density peaks.

➔

➔
Grid Labels
  • Input: Human density grid.
  • Mechanism: Clusters grid cells based on spatial proximity to centroids.
  • Output: Each grid cell is assigned a cluster ID. Agents inherit the ID of their current cell.
Agent-Based To Be Implemented

3. DBSCAN (Density-Based Spatial Clustering of Applications with Noise)

A density-based algorithm that clusters agents directly based on their spatial proximity to each other.

➔
DBSCAN (Points)
➔
Agent Cluster IDs
➔
Grid Mapping
  • Input: Individual agent (pos_x, pos_y) coordinates.
  • Mechanism: Groups agents into clusters if they are within eps distance and have at least min_pts neighbors.
  • Output: Each agent is assigned a cluster ID. Grid cells inherit the majority cluster ID of agents within them.

Framework Integration

To allow easy switching, we will utilize the cluster_store_t structure in mod_clustering.f95. The active algorithm is controlled by world%config%clustering_algorithm.

Algorithm ID Constant Name Mode
1 CLUSTERING_ALG_WATERSHED Grid → Grid
4 CLUSTERING_ALG_KMEANS_AUTO Grid → Grid
3 (Updated) CLUSTERING_ALG_DBSCAN Agent → Agent → Grid

Implementation Strategy

  1. Data Adapter: Create dbscan_agent_cluster in mod_kmeans.f95 to bridge the Agent-World and the raw DBSCAN algorithm.
  2. Grid Mapping: Implement a majority-voting mechanism to project agent-based clusters back onto the grid for visualization.
  3. Unified Dispatch: Refine mod_clustering.f95 to handle both grid-based and agent-based inputs gracefully.
